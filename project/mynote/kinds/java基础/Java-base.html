1.1计算机发展史

第一代计算机：（40年代中期到50年代末期） 电子管计算机，体积庞大，运算速度慢，功耗高，可靠性差，价格昂贵。应用领域军事和科学。

第二代计算机：（50年代末期到60年代中期） 晶体管计算机，体积缩小，运算速度提高，功耗降低，可靠性提高，运算速度提高，性能比第一代大大提高。开始向非数值方向发展，应用的领域主要是科学计算、企业信息管理。

第三代计算机：（60年代中期到70年代初期） 小规模集成电路计算机，运算速度显著提高，可靠性提高，价格降低。应用领域主要是文字处理，图像图形处理，基本上进入到经济、工业、科学等各个领域。

第四代计算机：（70年代至今） 大规模集成电路计算机新时代，硬件更加精化，软件更加丰富，走向家庭应用，面向社会。

1.2计算机应用

科学计算 一般用于国家来用于航天技术，地震预测，气象预报，武器制造，多使用超级计算机。

过程监控 利用计算机对工业生产工程中的某些信号自动进行检测，工厂流水线等。

信息管理（我们需要学习接触的） 信息管理是我们需要关注的一个领域，比如管理企业信息，erp，oa，ems等。

人工智能 开发一些具有人类特征的程序来模拟人的思维和动作，机器人等。

语言翻译 尚不成熟，有道词典，金山词霸等。

1.3计算机分类

超级计算机 通常是指最大，最快，最贵的计算机，主要用于，国防，武器制作，航天技术这些领域。

大型机 大型机主要应用于商业领域，如果银行电信行业，作为大型项目的服务器。

小型机 小型机是运行原理类似于PC，主要用于服务器，性能高。

微型机 PC（personal computer）个人电脑

1.4计算机系统

计算机系统由计算机硬件和软件两部分组成。硬件包括中央处理机、存储器和外部设备等；软件是计算机的运行程序和相应的文档。计算机系统具有接收和存储信息、按程序快速计算和判断并输出处理结果等功能。常见的系统有Windows，Linux等。

1.6计算机语言概述

计算机能识别的语言。 1.机器语言：由0和1组成的可以被电脑理解，人很难理解。

2.汇编语言：由汇编软件把汇编语言翻译成机器语言然后电脑可以理解，人可以理解。

3.高级语言：与人类的自然语言很接近，由高级语言的编译器来把高级语言翻译成机器语言，然后计算机来运行，java，c，c++,….

 

1.7java概述

1995年诞生于sun公司，2009被Oracle收购。 Java语言的特点：

  1. 简洁有效：面向对象，简单易学。 2.java跨平台：java写好的程序可以在不同平台来运行，一次编译处处运行。Java为每一个平台提供jvm（java virtual machine）。 3.适合分布式项目：java语言具有强大的，易于使用的互联网能力，非常适合大型的互联网分布式项目。

Java分支 1.Javase：（标准版）前身j2se,2005年后改成javase（我们的基础课程，必学） 2.Javame：（微型版）前身是j2me,2005年后改成javame，给移动设备做支持，目前基本上淘汰了。（不学） 3.Javaee：（企业版）前身j2ee，2005年后改成企业级应用技术开发（就业课程，必学）

1.10第一个java示例以及书写规范

文件名：Java文件是以.java为后缀； 类名：类名要和文件名一致，类名的首字母要大写； 方法：在类中写方法，方法体写在大括号里面，注意一定是成对出现，类和方法之间要有缩进，方法和方法体之间也要有缩进，缩进使用Tab键，不要使用空格。 分号：分号的出现代表本行代码结束。 大括号：代表界限，类的界限、方法的界限。

1.13标识符

1.在java语言中，对各种变量，方法和类等要素命名时所使用的字符序列就是标识符。 这些标识符有jdk定义好的，也有我们自己开发时候需要定义的。 2.java中标识符由26个英文字符大小写，数字0-9，符号_ $。注意：数字不能开头。 3.java中是严格区分大小写 4.在实际写程序的过程中定义标识符遵守“见名之意”.

标识符-代码规范

类名和接口的规范： 在上面的约束的命名规范的情况下，首字符要大写。如果由多个单词构成每个单词的首字符都大写。 XxxYyy 如：PersonService 变量名和方法名： 在上面的约束的命名规范的情况下，我们遵守单词首字符不要大写，如果由多个单词构成，从第二个单词起首字符都大写。这叫驼峰模式。 xxxYyyZzz 如：personDao 包名 在上面的约束的命名规范的情况下，都小写。 xxx.yyy.zzz 常量 所有字符都大写，多个单词之间使用下划线来分隔。 XXX_YYY_ZZZ

1.14关键字

Jdk定义的标识符就是关键字，我们不可以使用。所以我们定义标识符的时候一定要避开关键字。

1.16java中的注释

注释就是给程序说明的作用，我们要养成写注释的良好习惯，增强代码的可读性。Jvm运行时注释不会被解析。Jvm会跳过注释。

 

单行注释语法：//，注释代码中的一行，建议大家放在要注释的代码的上面 多行注释：/*….*/,通常用于方法内的多行或者类的多行注释 文档注释：语法/**..*/,用于类的说明和方法的说明
